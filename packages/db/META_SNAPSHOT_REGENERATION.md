# Migration System: Meta Snapshot Regeneration

> **Status:** Documentation — snapshots must be regenerated by running
> `drizzle-kit introspect` against a fully-migrated database.

## Problem (MIG-1)

The `packages/db/drizzle/meta/` directory contains only `_journal.json` —
no `*_snapshot.json` files. Drizzle-kit normally creates a snapshot file
for each migration so that `drizzle-kit generate` can diff against the
previous schema state. Without snapshots, `drizzle-kit generate` cannot
determine what changed and will generate migrations that re-create
existing tables, prompting the interactive "created or renamed from
another table?" dialog.

This is why `run_drizzle.py` (now removed) existed — it auto-accepted
those prompts via `pexpect`, which always accepted the default and
potentially created incorrect migrations.

## Solution

### Regenerating Snapshots

To regenerate all snapshot files:

1. Start a fresh PostgreSQL instance (or use Docker):
   ```bash
   docker run --rm -d --name hamafx-pg \
     -e POSTGRES_PASSWORD=postgres \
     -p 5432:5432 \
     pgvector/pgvector:pg16
   ```

2. Set `DATABASE_URL` and apply all migrations:
   ```bash
   export DATABASE_URL="postgresql://postgres:postgres@localhost:5432/postgres"
   pnpm --filter @hamafx/db run migrate:apply
   ```

3. Run drizzle-kit introspect to generate snapshots:
   ```bash
   cd packages/db
   npx drizzle-kit introspect
   ```

4. Verify snapshot files were created:
   ```bash
   ls drizzle/meta/*_snapshot.json
   ```

5. Commit the regenerated snapshot files.

### Future Migrations

For all future migrations, use `drizzle-kit generate --custom` and write
the migration SQL by hand. This avoids the interactive prompts entirely
and gives full control over the migration content.

```bash
cd packages/db
npx drizzle-kit generate --custom --name=<descriptive_name>
```

Then manually edit the generated SQL file and add the entry to
`_journal.json` if drizzle-kit doesn't do so automatically.

## Verification

After regenerating snapshots, verify that `drizzle-kit generate` no longer
prompts with "created or renamed from another table?" when run without
schema changes. If it does, the snapshots are incomplete.
