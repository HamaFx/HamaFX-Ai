
# Steering: web UI

When working in `apps/web/**`:

1. **Mobile-first** for iPhone 14 Pro Max (430×932). All layouts designed for phone first; desktop is an enhancement.
2. Use server components by default. `"use client"` only where you need state, events, or browser-only APIs.
3. Use semantic Tailwind tokens (`bg-bg-elev-1`, `bg-bg-elev-2`, `bg-bg-elev-3`, `text-fg-muted`, `text-bull`, `text-bear`, `border-divider`) — never raw colors.
4. Interactive elements have ≥ 44×44 tap targets on mobile and visible focus rings always.
5. Numbers (prices, deltas, P/L, R-multiples) use `.tabular-nums` utility for consistent column alignment.
6. Pages must define **loading**, **empty**, **error**, and **stale** states. Stale = `isFetching && !isLoading` from TanStack Query — surface via `<StaleIndicator>`.
7. URL state via `nuqs` for selected symbol, timeframe, indicator overlays — never local-only.
8. Data fetching: TanStack Query for client cache. Server components prefetch via the same adapters.
9. Charts: use `lightweight-charts` via the wrapper in `components/chart/`. Never instantiate it directly in pages.

## Navigation (Phase 6)

10. **Single global `<NavDrawer/>`** mounted once in `(app)/layout.tsx`. Both `<TopBar/>` and `<ChatTopBar/>` open it via `<NavTrigger/>` which calls `setOpen(true)` from `useNavDrawer()`. Never mount a second drawer instance.
11. **Global `<TopBar/>` is suppressed on `/chat`** — the chat surface renders its own `<ChatTopBar/>`. Suppression lives inside `<TopBar/>` itself; do not re-add a chat-specific TopBar at the layout level.
12. **There is no bottom navigation.** It was retired in Phase 6. The `/more` route was removed.

## Design system primitives — use the canonical one, don't roll new

13. **Icons**: `lucide-react` exclusively. No inline SVGs in components (the only intentional exception is `<AmbientBackground intensity="vivid">` on `/login`). Sizes: `size-3.5` (inline), `size-4` (buttons/cards), `size-5` (nav), `size-6` (FAB/headers).
14. **Motion**: `motion/react` with `LazyMotion + domAnimation`. Mounted via `<MotionRoot>` in `(app)/layout.tsx`. Use `m.button`/`m.div` (not `motion.button`) so the lazy bundle is smaller.
15. **Bottom sheets**: `<Drawer>` (vaul wrapper). Centered dialogs are reserved for modal-blocking confirmations only — and use `<ConfirmDrawer>` / `useConfirm()`, never `window.confirm()`.
16. **Floating action buttons**: `<Fab>`. Position is driven by the `--fab-bottom` CSS token.
17. **Toasts**: `sonner` via `toast.success(...)` / `toast.error(...)`. Replace inline status strings.
18. **Animated numbers**: `<AnimatedNumber>` (state-bound subscription with `restDelta` so the spring stops scheduling work after settle).
19. **Stat cards**: `<StatCard>` with optional sparkline.
20. **Segmented controls**: `<Segmented>` — `variant="gradient" | "solid" | "tone"`. Do not roll new ad-hoc segment groups.
21. **Tooltips**: `<Tooltip>` for icon-only buttons. Always pair with `aria-label` on the trigger.
22. **Skeletons**: `<Skeleton>` / `<SkeletonCard>` (uses the `.shimmer` keyframe). Do not use bare `animate-pulse` for new placeholders.
23. **Empty states**: `<EmptyState>` with `tone="brand"` (CTA-led) or `tone="muted"` (informational).
24. **Toggles**: `<Switch>` (CSS-only).
25. **Buttons**: `<Button>` with sizes `sm (40) | md (48) | lg (56)` and variants `primary | secondary | ghost | danger | success`.
26. **Inputs**: `<Input>` (h-12, text-base — kills iOS auto-zoom).

## Theme tokens

27. **Layout heights** live as CSS custom properties on `:root`: `--topbar-h`, `--fab-bottom`, `--toast-bottom`. Sticky in-page headers reference `var(--topbar-h)`.
28. **Gradients & branded shadows** are CSS variables: `--gradient-brand`, `--gradient-danger`, `--gradient-brand-soft`, `--gradient-accent`, `--shadow-brand-press`, `--shadow-brand-press-strong`, `--shadow-danger-press`, `--shadow-inset-edge`, `--shadow-inset-edge-soft`. Reference via `style={{ backgroundImage: 'var(--gradient-brand)' }}`. Don't hard-code OKLCH gradient stops in component code.
29. **Surfaces are pure neutral grayscale** (oklch hue 0 chroma 0). Don't reintroduce blue tint.
30. **8-pt grid** spacing tokens (`--space-1..10`) and `--touch-min: 44px` available for JS calc.

## Stability helpers

31. Use `paint-isolated` (CSS `contain: layout paint`) on full-bleed surfaces to prevent flashes during route transitions.
32. Use `no-overscroll` (`overscroll-behavior: contain`) on scrollable containers so iOS Safari doesn't rubber-band past the chrome.
33. Initial-mount scroll uses instant `scrollTop = scrollHeight`. Never `behavior: 'smooth'` on first paint.

## Safe-area handling

34. Root viewport meta has `viewportFit: 'cover'` (in `app/layout.tsx`) so `env(safe-area-inset-*)` returns Dynamic Island / home-indicator values.
35. Tailwind utilities `pt-safe`, `pb-safe`, `bottom-safe`, `top-safe` map to `padding-* / bottom: env(safe-area-inset-*)`.
36. Fixed-position elements (FAB, toasts) clear the home indicator via the layout tokens above.

## Accessibility

37. `<html lang>` is set to `"en"` until a locale switch ships. Voice input reads `navigator.language` for STT only — it does not change layout direction.
38. Skip-to-main-content link (`<SkipToContent/>`) is mounted as the first focusable element in `(app)/layout.tsx`. Pages must include `<main id="main-content">`.
39. **Color-not-only**: red/green/amber must always be paired with an icon or shape (e.g. `▲ beat` / `▼ miss`, BellRing/BellOff/Bell on alert states, ▲/■/• on importance dots).
40. **Reduced motion**: respected via the OS media query AND a user-forced override (`data-reduce-motion="force"` on `<html>`, set by the Preferences card).

## Local preferences

41. Local-storage keys use the `hamafx:` prefix (e.g. `hamafx:news:bookmarks`, `hamafx:prefs`). Cross-tab sync via the `storage` event.
42. Any new local preference adds a row to `<PreferencesCard/>` and a clear-affordance to `<DataCard/>` in `apps/web/src/app/(app)/settings/_components/`.
