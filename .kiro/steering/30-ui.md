
# Steering: web UI

When working in `apps/web/**`:

1. **Mobile-first** for iPhone 14 Pro Max (430×932). All layouts designed for phone first; desktop is an enhancement.
2. Use server components by default. `"use client"` only where you need state, events, or browser-only APIs.
3. Use semantic Tailwind tokens (`bg-bg-elev-1`, `bg-bg-elev-2`, `bg-bg-elev-3`, `text-fg-muted`, `text-bull`, `text-bear`, `border-divider`) — never raw colors.
4. Interactive elements have ≥ 44×44 tap targets on mobile and visible focus rings always.
5. Numbers (prices, deltas, P/L, R-multiples) use `.tabular-nums` utility for consistent column alignment.
6. Pages must define **loading**, **empty**, **error**, and **stale** states. Stale = `isFetching && !isLoading` from TanStack Query — surface via `<StaleIndicator>`.
7. URL state via `nuqs` for selected symbol, timeframe, indicator overlays — never local-only.
8. Data fetching: TanStack Query for client cache. Server components prefetch via the same adapters.
9. Charts: use `lightweight-charts` via the wrapper in `components/chart/`. Never instantiate it directly in pages.

## Design system primitives

10. **Icons**: `lucide-react` exclusively. No inline SVGs in components (the only exception is `<AmbientBackground>` which renders the `feTurbulence` noise filter once for the whole app). Sizes: `size-3.5` (inline), `size-4` (buttons/cards), `size-5` (nav), `size-6` (FAB/headers).
11. **Motion**: `motion/react` with `LazyMotion + domAnimation`. Mounted via `<MotionRoot>` in `(app)/layout.tsx`. Use `m.button`/`m.div` (not `motion.button`) so the lazy bundle is smaller. All animations respect `prefers-reduced-motion` automatically via `MotionConfig reducedMotion="user"`.
12. **Bottom sheets**: use `<Drawer>` (vaul wrapper at `components/ui/drawer.tsx`) for context-shifting actions on mobile (alert/journal create forms, chart overlay toggles). Centered dialogs are reserved for modal-blocking confirmations only — and use `<ConfirmDrawer>` / `useConfirm()` from `components/ui/confirm-drawer.tsx`, never native `window.confirm()`.
13. **Floating action buttons**: use `<Fab>` from `components/ui/fab.tsx`. Position is driven by the `--fab-bottom` CSS token (= `--bottom-nav-h + 8px + safe-area-inset-bottom`).
14. **Toasts**: use `sonner` via `toast.success(...)` / `toast.error(...)` for write confirmations. Replace inline status strings.
15. **Animated numbers**: use `<AnimatedNumber>` for live-updating prices so digit changes spring smoothly. The component uses a state-bound subscription (not `useTransform`) so the spring stops scheduling work after it settles.
16. **Stat cards**: use `<StatCard>` with optional sparkline for numeric summaries.
17. **Segmented controls**: use `<Segmented>` from `components/ui/segmented.tsx` — single primitive with `variant="gradient" | "solid" | "tone"`. Do not roll new ad-hoc segment groups.
18. **Tooltips**: use `<Tooltip>` from `components/ui/tooltip.tsx` for icon-only buttons. Always pair with `aria-label` on the trigger so SR users get the same info.
19. **Skeletons**: use `<Skeleton>` / `<SkeletonCard>` from `components/ui/skeleton.tsx`. They use the `.shimmer` keyframe defined in `globals.css`. Do not use bare `animate-pulse` for new placeholders.

## Theme tokens

20. **Layout heights** live as CSS custom properties on `:root` so JS calc/sticky offsets reference them once: `--topbar-h`, `--bottom-nav-h`, `--fab-bottom`, `--toast-bottom`. Update one place to shift the whole shell.
21. **Gradients & branded shadows** are CSS variables: `--gradient-brand`, `--gradient-danger`, `--gradient-brand-soft`, `--gradient-accent`, `--shadow-brand-press`, `--shadow-brand-press-strong`, `--shadow-danger-press`, `--shadow-inset-edge`. Reference via `style={{ backgroundImage: 'var(--gradient-brand)' }}`. Don't hard-code OKLCH gradient stops in component code.

## Safe-area handling

22. Root viewport meta has `viewportFit: 'cover'` (in `app/layout.tsx`) so `env(safe-area-inset-*)` returns Dynamic Island / home-indicator values.
23. Tailwind utilities `pt-safe`, `pb-safe`, `bottom-safe`, `top-safe` map to `padding-* / bottom: env(safe-area-inset-*)`.
24. Fixed-position elements (FAB, BottomNav, toasts) clear the home indicator via the layout tokens above.

## Bottom nav (mobile)

`[ Chat ]   [ Chart ]   [ News ]   [ Calendar ]   [ More ]` — uses lucide icons with a sliding `motion.div layoutId` indicator behind the active item.

| `text-xs` | 12px | Secondary meta |

| `text-base` | 16px | Primary body |
| `text-lg` | 18px | Section headings, prices |

| Token | Use |
|---|---|
| `bg-bg` | Root canvas |
| `bg-bg-elev-1` | Cards, list items, composer |
| `bg-bg-elev-2` | Buttons, pills, segmented controls, drawer body |
| `bg-bg-elev-3` | Active states inside elevated surfaces (selected pill bg) |

## Accessibility

25. `<html lang>` is set to `"en"` until a locale switch ships. Voice input reads `navigator.language` for STT only — it does not change layout direction.
26. Skip-to-main-content link is mounted as the first focusable element in `(app)/layout.tsx` (`<SkipToContent>`). Pages must include `<main id="main-content">` for the link to land.
27. Color-not-only rule: red/green/amber must always be paired with an icon or shape (e.g. `▲ beat` / `▼ miss`, BellRing/BellOff/Bell on alert states).
