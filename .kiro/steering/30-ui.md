# Steering: web UI

When working in `apps/web/**`:

1. **Mobile-first** for iPhone 14 Pro Max (430Ă—932). All layouts designed for phone first; desktop is an enhancement.
2. Use server components by default. `"use client"` only where you need state, events, or browser-only APIs.
3. Use semantic Tailwind tokens (`bg-bg-elev-1`, `bg-bg-elev-2`, `bg-bg-elev-3`, `text-fg-muted`, `text-bull`, `text-bear`, `border-divider`) â€” never raw colors.
4. Interactive elements have â‰Ą 44Ă—44 tap targets on mobile and visible focus rings always.
5. Numbers (prices, deltas, P/L, R-multiples) use `.tabular-nums` utility for consistent column alignment.
6. Pages must define **loading**, **empty**, **error**, and **stale** states.
7. URL state via `nuqs` for selected symbol, timeframe, indicator overlays â€” never local-only.
8. Data fetching: TanStack Query for client cache. Server components prefetch via the same adapters.
9. Charts: use `lightweight-charts` via the wrapper in `components/chart/`. Never instantiate it directly in pages.

## Design system primitives (Phase 5)

10. **Icons**: `lucide-react` exclusively. No inline SVGs in components. Sizes: `size-3.5` (inline), `size-4` (buttons/cards), `size-5` (nav), `size-6` (FAB/headers).
11. **Motion**: `motion/react` with `LazyMotion + domAnimation`. Mounted via `<MotionRoot>` in `(app)/layout.tsx`. Use `m.button`/`m.div` (not `motion.button`) so the lazy bundle is smaller. All animations respect `prefers-reduced-motion` automatically via `MotionConfig reducedMotion="user"`.
12. **Bottom sheets**: use `<Drawer>` (vaul wrapper at `components/ui/drawer.tsx`) for context-shifting actions on mobile (alert/journal create forms, chart overlay toggles). Centered dialogs are reserved for modal-blocking confirmations only.
13. **Floating action buttons**: use `<Fab>` from `components/ui/fab.tsx`. Positioned `bottom: calc(80px + env(safe-area-inset-bottom))` to clear BottomNav + home indicator.
14. **Toasts**: use `sonner` via `toast.success(...)` / `toast.error(...)` for write confirmations. Replace inline status strings.
15. **Animated numbers**: use `<AnimatedNumber>` for live-updating prices so digit changes spring smoothly.
16. **Stat cards**: use `<StatCard>` with optional sparkline for numeric summaries.

## Safe-area handling

17. Root viewport meta has `viewportFit: 'cover'` (in `app/layout.tsx`) so `env(safe-area-inset-*)` returns Dynamic Island / home-indicator values.
18. Tailwind utilities `pt-safe`, `pb-safe`, `bottom-safe`, `top-safe` map to `padding-* / bottom: env(safe-area-inset-*)`.
19. Fixed-position elements (FAB, BottomNav, toasts) clear the home indicator via safe-area bottom padding.
