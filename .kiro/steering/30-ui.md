# Steering: web UI

When working in `apps/web/**`:

1. **Mobile-first.** All layouts must be designed for a phone first; desktop is an enhancement.
2. Use server components by default. `"use client"` only where you need state, events, or browser-only APIs.
3. Use semantic Tailwind tokens (`bg-bg-elev-1`, `text-fg-muted`, `text-bull`, `text-bear`) — never raw colors.
4. Use `tailwind-variants` for any component with > 2 visual variants.
5. Interactive elements have ≥ 44×44 tap targets on mobile and visible focus rings always.
6. Numbers (prices, deltas, P/L) use `font-feature-settings: "tnum","cv01"` (`.tabular-nums` utility).
7. Pages must define **loading**, **empty**, **error**, and **stale** states.
8. URL state via `nuqs` for selected symbol, timeframe, indicator overlays — never local-only.
9. Data fetching: TanStack Query for client cache. Server components prefetch via the same adapters.
10. Charts: use `lightweight-charts` via the wrapper in `components/chart/`. Never instantiate it directly in pages.
